# FASE 2 — GESTIÓN DE SERVICIOS (CORE)

## 🎯 Objetivo
Implementar el flujo completo del servicio según el SRS.

## 📚 Fuente de verdad
SRS:
- RF-02 Gestión de Servicios
- RN-04 Historial obligatorio
- RN-05 Un solo mensajero por servicio
- Estados del servicio
- CU-01 a CU-07

## 🧩 Alcance

### Funcionalidades
- Crear servicio
- Asignar servicio
- Cambiar estados
- Cancelar servicio
- Consultar servicios
- Historial de estados

## 🔁 Estados válidos

PENDIENTE → ASIGNADO → ACEPTADO → EN_RUTA → ENTREGADO

Cancelación:
- PENDIENTE → CANCELADO
- ASIGNADO → CANCELADO

## 🗃️ Entidades
- servicios
- historial_estado_servicios
- cliente
- mensajero

## 🔌 Endpoints

- POST /services
- POST /services/:id/assign
- POST /services/:id/status
- POST /services/:id/cancel
- GET /services
- GET /services/:id

## ⚙️ Reglas clave

- Validar transiciones de estado
- Registrar historial en cada cambio
- Solo 1 mensajero por servicio
- Validar pertenencia a empresa

## ✅ Definition of Done

- Flujo completo de servicio funcional
- Máquina de estados validada
- Historial registrado correctamente
- No se permiten estados inválidos

## 🧠 Estado final del sistema

- Sistema puede operar servicios reales
- Flujo logístico completamente funcional
- Base del negocio implementada
